Ruby Gems

12 September 2015

Gems are libraries made up out of Ruby code and packaged up to run on any platform that Ruby runs on. Each gem has a name, version and contains code, documentation and gemspec. The gem can be easily downloaded and installed on your computer and will be available for you to use in every program you write. Rubygems.org is the most popular repository of “gems” for the Ruby language.

An example of a Ruby gem is Byebug which allows you to see what is going on inside a Ruby program while it executes and offers many of the traditional debugging features such as:

  • Stepping - running your program one line at a time.
  • Breaking - pausing the program at some event or specified instruction, to examine the current state.
  • Evaluating: Basic read–eval–print loop (REPL) functionality
  • Tracking: Keeping track of the different values of your variables or the different lines executed by your program.

The below example shows you how to use byebug in code. Here's method that takes a string and returns an array whose keys are all the downcased words in the string, and values are number of times these words were seen. To use the byebug gem, you will need to require it at the beginning of the method. You can then insert 'byebug' where you want to track your code.

require 'byebug'

def word_count(string)
  separate_words = string.downcase.split
  words_counted = Hash.new(0)
  separate_words.each { |word| 
    words_counted.store(word, words_counted[word] + 1) }
  words_counted
  byebug
end

word_count "The dog and the cat"

Dangers of Ruby gems

Some of the dangers of using gems is you are essentially trusting that the gem works as well as the it's developer. Asides from well-known and trusted developers, you don't essentially know who they are. As gems can run code on install, the risk is that gems can steal the unencrypted Rubygems.org credentials which can then be used to modify the compromised user's gems and spread malicious code.

It is also impossible to notify a gem user of compromised gems. When a gem developer’s credentials are compromised it is hard to notify anyone who uses any of the gems published by the developer of the situation.
